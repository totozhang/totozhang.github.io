VirtualBox Network Configuration

Posted on 2015-12-24 | In Network tools

VirtualBox is a powerful x86 and AMD64/Intel64 virtualization software (VBox for short), which can be used to construct kinds of debug or test network environments, to capture or analyse the packets in the test network. Sometimes, for the sake of security, some virtual hosts run parallely on the same physical platform to isolate each other, decrease complexity. Sometimes, Vbox is used as hack tools to hide from tracking. All these usages are based on the network configuration of Vbox itself. This article is about the network configurations and the topology of each configuration. 既然涉及到网络，那就应该从VBox为用户提供的硬件资源开始说起。

网络适配器

VBox为每台客户机提供8块PCI以太网卡，其中，4块通过GUI配置，4块需要通过命令行进行配置（实际上，8块网卡都能通过命令行进行配置）。

网络选项

VBox一共提供了7种可选网络类型以满足各种不同的使用需求。下面详细说明VBox提供的各类可选择的网络类型。

类型1: Not attached

在Not attached情况下，客户机操作系统认为计算机中有网卡存在，但是网卡中没有插入网线。如果在使用过程中需要对客户机进行插拔网线的测试操作，可以选用此选项。实际上在Advanced设置里边，有个选项叫做Cable Connected，更适合做插拔网线的操作。

类型2: Network Address Translation (NAT)

NAT选项为VBox默认的网络类型选项。使用NAT选项时，VBox为每台客户机单独提供一个NAT路由器，客户机和宿主机拓扑为,

如果宿主机接入了Internet，那么客户机可以访问Internet，进行网页浏览，邮件收发等。从物理网络看来，来自各个客户机的数据包全部来自VBox进程。同时也说明了NAT网络可嵌套使用，我个人认为这使得IPv6的推进更加艰难。

此时，如果用户需要从物理网络访问VBox中的某一台客户机，则需要进行Port forwarding配置。

类型3: NAT Network (实验阶段)

2015-12-24-network-topology-in-virtualbox/VBox3.png VBox为客户机上设置为NAT Network选项的网络适配器提供同一NAT路由器。使用这个选项时，宿主机和客户机类似家用NAT路由器的拓扑关系。

NAT Network跟NAT选项区别在于，在NAT Network中，客户机在同一个私有局域网内，共用同一个出口网关。

类型4: Bridged Networking

桥接模式，宿主机和客户机相当于彼此桥接，可以理解为所有主机都在同一个以太网内。此时客户机相当于直接接入物理网络的计算机，如果物理网络中有DHCP服务器，客户机也自动获取到IP地址。理论上，在桥接网络中，客户机和VBox本身都可对外提供DHCP服务，但这非常可能与网络中已有的DHCP服务器冲突，所以尽量不要这么做。

Bridged Networking模式

如果宿主机是Multihomed，桥接的网络适配器需要指定。

虽然这种模式下，各网卡适配器都可以做为桥接设备，但是这种模式仅对以太网有效，包括有线和无线以太网(WiFi)。值得注意的是，使用桥接相当于多台计算机同时接入到物理网络中，所以对其可能造成冲击，比如其中一台客户机上运行了DHCP服务。在运维项目中，调试一些云级服务器时，常用这种模式来运行Jump Start服务，同时这种模式也是一些所谓黑客的首选。

类型5: Internal Networking

在网络应用的开发，调试，或测试过程中，都需要进行网络抓包分析。很多工程师会首选前边介绍的Bridged模式，这种情况下，使用Bridged模式至少带来两个问题，首先，宿主机要接入物理网络，其次，抓包分析过程中，由于物理网络中有其它各类设备，各种无关类型的包在里边穿梭往来，给工程师分析数据带来了不必要的麻烦。这就好像学习驾驶过程中，一般都先在练车场地中练习开车，而不要直接去车辆很多的市区主干道。这种模式中，内部网络与宿主机之间没有任何联系，用户得到的是一个纯净的网络。

类型6: Host-only Networking

宿主机调试网络应用程序，难道还非得发到内部网络的某台虚拟机上调试吗？不必，Host-only Networking选项也可以作为一种选择。Host-only模式跟Internal模式基本类似，但是Host-only可以由用户自行建立这个内部网络，并且宿主机自动加入到新建立的内部网络中。在这种模式中，VBox是否提供DHCP服务是可选的。
